# Основные функции

1) Написание пользователем букв, которые расположены на пути.
2) Сервер -> Игрок создает лобби и любой участник может присоединиться к игре.
3) Кампания, где игрок может потренировать основные клавиши.
4) Бесконечный режим, где пользователь вводит буквы. А появление букв ускоряется каждые пару минут на клавиш/сек или за каждую ошибку пользователя. 
5) Турнир. Из 8 человек. каждые 50 символов, количество уменьшается в 2 раза.

# Ограничения на процесс разработки

- **Модель разработки**: водопадная
- **Состав команды**: Проект выполняется командой из 4 участников.
- **Платформа разработки**:
	  - Язык разработки: C#, С++
	  - Кроссплатформенная среда разработки игр: Unity
	  - Платформенная поддержка:
		- Компилирование: Windows
		- Отладка: Windows
		- Выполнение: Windows		
- **Временное ограничение**: Проект должен быть приведен к минимальному рабочему состоянию, протестирован и аттестован к 24 декабрю 2025 года.
- **Техническое ограничение**: личные ноутбуки, персональные компьютеры.
- **Бизнес ограничение**: Бюджет проекта ограничен, что заставляет использовать в проекте либо бесплатные спрайты/модели/текстуры из общего доступа, либо создавать их самим. 
- **Ограничения по навыкам**: Все участники имеют хороший уровень языков программирования(C#, C++). Большая часть участников не имеет опыта разработки игр, что потребует дополнительного изучения кроссплатформенной среды разработки компьютерных игр Unity.

# Формы отчетности и взаимодействия

- Собрания в очной форме проводятся каждую неделю по средам в 17:00-18:35.
- Собрания в дистанционной форме проводятся по согласованию всех участников каждую неделю.
- Совместная работа ведется каждый день в вечернее время в течении не менее двух часов. Для совместной работы используются социальный мессенджер Telegramm.
- В качестве центрального хранилища кода и индивидуальных отчетов используется веб-сервис для хостинга IT-проектов GitHub. Центральный репозиторий располагается по ссылке: https://github.com/kinkradProg/Project-TPPO
- Разработчики сами ведут индивидуальные отчеты и выкладывают их в репозиторий проекта в папку *Индивидуальные отчеты*
- Раз в неделю формируется общий отчет о состоянии проекта(В воскресенье до 20:00) и проделанной работы, который отправляется инструктору. Ответственный: Инюшин Н.В. или Бедорев А.А.
