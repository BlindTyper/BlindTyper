# Глоссарий терминов

***Пользователь*** - человек, который использует приложение.
***Игровой процесс*** - запуск пользователем уровня.
***Уровень*** - часть приложения, где пользователю необходимо нажимать на правильные клавиши на клавиатуре.
***Кампания*** - часть приложения, где пользователь поочередно проходит различные уровни для тренировки различных клавиш.
***Бесконечный режим 1*** - часть приложения, где пользователь нажимает на клавиши отображаемые на мониторе пока ему не надоест.
***Бесконечный режим 2*** - часть приложения, где пользователь нажимает на клавиши отображаемые на мониторе, пока он не проиграет. В процессе скорость с которой необходимо нажимать на клавиши увеличивается.
***Онлайн игра*** - часть приложения, где пользователь проходит уровень с другими пользователями. 
***Лобби*** - часть приложения, куда пользователи присоединяются для участия в онлайн игре.
***Рейтинг*** - топ из игроков, которые быстрее всего справляются с уровнями или имеют наибольшую скорость печати в минуту.
***Смерть пользователя*** - смерть аватара пользователя в игре, когда он не успел нажать на правильную клавишу, а время или жизни закончились.
***Жизни пользователя*** - количество промахов, которое могут допустить пользователи, пока не наступит смерть персонажа.
***Отобразить*** - отрисовать на экране все спрайты связанные с тем объектом, который необходимо отобразить на экране.
***Спрайт*** -  графический объект в компьютерной графике или же картинка объекта.

# Первичный список требований
Каждое из требований к проектируемой системе, имеет один из следующих приоритетов
- H - Высокий
- M - Средний
- L - Низкий
Категория определяет приоритет выполнения требований на последующих стадиях. Первыми реализуются требования с высоким приоритетом, вторыми - со средним, третьими - с низким. Предполагается, что требования с высоким приоритетом будут выполнены даже в худших условиях.

## Функциональные требования:
1) Главное меню:
	1) Пользователь может зайти в меню кампании, а приложение должно отобразить его (H)
	2) Пользователь может зайти в меню бесконечной игры, а приложение должно отобразить его (M)
	3) Пользователь может зайти в онлайн игру, а приложение должно отобразить его (M)
	4) Пользователь может зайти в меню рейтинг, а приложение должно отобразить его (L)
	5) Пользователь может зайти в меню настроек, а приложение должно отобразить его (L)
	6) Пользователь может выйти из приложения, а приложение должно закрыться (H)
2) Меню настроек:
	1) Пользователь может менять с помощью ползунка громкость звука, а приложение должно его изменить (L)
	2) Пользователь может поменять разрешение экрана с помощью выпадающего списка, а приложение должно его изменить (L)
	3) Пользователь может поменять тему приложения, а приложение должно его изменить (L)
	4) Пользователь может вернуться в главное меню нажав на соответствующую кнопку, а приложение должно отобразить его (L)
3) Меню кампании:
	1) Пользователь может запустить уровень игры, который ему доступен, а приложение должно запустить его (H)
	2) Приложение должно открыть новый уровень при прохождении прошлого (H)
	3) Пользователь может вернуться в главное меню нажав на соответствующую кнопку, а приложение должно отобразить его (H)
4) Меню бесконечной игры:
	1) Пользователь может зайти в режим бесконечной печати, а приложение должно запустить его (M)
	2) Пользователь может зайти в режим скоростной печати, а приложение должно запустить его (M)
	3) Пользователь может выйти в главное меню, а приложение должно отобразить его (M)
5) Меню онлайн игры:
	1) Пользователь может создать лобби, а приложение должно создать его (H)
	2) Пользователь создатель лобби может настроить его, а приложение должно поменять настройки лобби (H)
	3) Пользователь может присоединиться в чужое лобби, а приложение должно подключить пользователя к лобби (M)
	4) Пользователь может запустить игровой процесс, а приложение должно запустить его для всех пользователей лобби (H)
6) Игровой процесс:
	1) Игрок может печатать буквы с экрана, а приложение должно уничтожать их (H)
	2) Игровая сессия должна закончиться при вводе последнего символа уровня (H)
	3) Игровая сессия должна закончиться при смерти персонажа (M)
	4) Игрок может убивать других игроков (M)
	5) Игрок может приостановить сессию, а приложение должно приостановиться (H)
	6) Игрок может возобновить сессию, , а приложение должно возобновиться (H)
	7) В случае выхода из игровой сессии прогресс игрока на уровне должен сброситься (H)
7) Хранение и обработка данных локально
	1) Приложение должно сохраненить прогресс пользователя (H)
	2) Приложение должно загрузить прогресс пользователя (H)
8) Аккаунт пользователя локально
	1) Приложение должно создать аккаунт пользователя (Н)
	2) Приложение должно авторизировать аккаунт пользователя (Н)
9) Аудио составляющая приложения:
	1) Звуки уничтожения блоков должны проигрываться
	2) Звуки заднего фона должны проигрываться
	3) Звуки смерти пользователя должны проигрываться
	
## Ограничения
1) Приложение управляется только с помощью мыши и клавиатуры
2) Пользовательский интерфейс на русском языке
3) Графическая часть приложения выполнена с помощью спрайтов
4) Серверная часть связывается с базой данных приложения

# Критерии аттестации системы

Для прохождения аттестации системы разработчики должны проверить:
1) Работоспособность приложения
	1) Все элементы UI части приложения должны отображаться корректно
	2) Все панели и кнопки работают корректно.
2) Тестирование приложения для пользователя
	1) Возможность зайти в различные режимы приложения.
	2) Профиль пользователя должен корректно отображать, а пользователи могут изменять данные о себе.
	3) Возможность соединения с сервером путем создания лобби в онлайн игре и присоединения к другим лобби.
3) 

Аттестация проводится разработчиками приложения и тестировщиками до сдачи работы. Аттестация считается пройденной, если:
1) Все функциональные требования будут реализованы, а также выполняются корректно.
2) Все ограничения системы будут соблюдены, а также работают правильно.
3) Пользовательский интерфейс адаптивен и работает.
4) Не выявлено критических ошибок, влияющих на работу приложения.
5) Успешно пройден сценарий аттестации.

Сценарий аттестации системы:
1) Пользователь устанавливает приложение себе на устройство.
2) Пользователь запускает приложение с помощью ярлыка.
3) Пользователь попадает в главное меню.
4) Пользователь нажимает на кнопку "Настройки". Попадает в меню "Настройки".
5) Пользователь меняет разрешение экрана. Разрешение изменяется.
6) Пользователь меняет громкость приложения. Громкость изменяется.
7) Пользователь нажимает на кнопку "Главное меню". Попадает в меню "Главное меню".
8) Пользователь нажимает на кнопку "Рейтинг". Попадает в меню "Рейтинг".
9) Приложение отображает рейтинг самых быстрых пользователей.
10) Пользователь нажимает на кнопку "Главное меню". Попадает в меню "Главное меню".
11) Пользователь нажимает на кнопку "Кампания". Попадает в меню "Кампании".
12) Приложение отображает уровни. Открыт только первый, другие считаются закрытыми.
13) Пользователь запускает первый уровень "Глава 1: Обучение".
14) Пользователь успешно завершает первый уровень.
15) Приложение открывает второй уровень "Глава 2: ва ол".
