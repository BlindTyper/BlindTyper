# Первичный список требований
Каждое из требований к проектируемой системе, имеет один из следующих приоритетов
- H - Высокий
- M - Средний
- L - Низкий
Категория определяет приоритет выполнения требований на последующих стадиях. Первыми реализуются требования с высоким приоритетом, вторыми - со средним, третьими - с низким. Предполагается, что требования с высоким приоритетом будут выполнены даже в худших условиях.

## Функциональные требования:
1) Меню авторизации:
	1) Пользователь может зайти в меню авторизации(H)
		1) Пользователь может зайти в подменю восстановления(M)
		2) Пользователь может авторизоваться(H)
		3) Пользователь может перейти в меню регистрации(L)z
	2) Пользователь может зайти в меню восстановления
		1) Пользователь может выслать письмо восстановления себе на почту(M)
		2) Пользователь может ввести новый пароль после подтверждения.(H)
		3) Пользователь может вернуться к авторизации.(M)
	3) Пользователь может зайти в меню регистрации(H)
		1) Пользователь может зарегистрироваться(H)
		2) Пользователь может перейти в меню авторизации(L)
2) Главное меню:
	1) Пользователь может зайти в меню кампании (H)
	2) Пользователь может зайти в меню бесконечной игры (M)
	3) Пользователь может зайти в онлайн игру (M)
	4) Пользователь может зайти в меню рейтинг (L)
	5) Пользователь может зайти в меню настроек (L)
	6) Пользователь может зайти в меню Профиля (M)
	7) Пользователь может выйти из приложения (H)
3) Меню настроек:
	1) Пользователь может менять с помощью ползунка громкость звука (L)
	2) Пользователь может поменять разрешение экрана с помощью выпадающего списка (L)
	3) Пользователь может поменять тему приложения(L)
	4) Пользователь может вернуться в главное меню нажав на соответствующую кнопку (L)
4) Меню кампании:
	1) Пользователь может запустить уровень игры, который ему доступен (H)
	2) Приложение может открыть новый уровень при прохождении прошлого (H)
	3) Пользователь может вернуться в главное меню нажав на соответствующую кнопку (H)
5) Меню бесконечной игры:
	1) Пользователь может зайти в режим бесконечной печати (M)
	2) Пользователь может зайти в режим скоростной печати (M)
	3) Пользователь может выйти в главное меню (M)
6) Меню онлайн игры:
	1) Пользователь может создать лобби (H)
	2) Пользователь создатель лобби может настроить его (H)
	3) Пользователь может присоединиться в чужое лобби (M)
	4) Пользователь может запустить игровой процесс (H)
7) Игровой процесс:
	1) Игрок может печатать буквы с экрана (H)
	2) Игровая сессия может закончиться при вводе последнего символа уровня (H)
	3) Игровая сессия может закончиться при смерти персонажа (M)
	4) Игрок может убивать других игроков (M)
	5) Игрок может приостановить сессию (H)
	6) Игрок может возобновить сессию (H)
	7) В случае выхода из игровой сессии прогресс игрока на уровне сбрасывается (H)
8) Хранение и обработка данных
	1) Сохранение прогресса пользователя (H)
	2) Загрузка прогресса пользователя (H)
	3) Обновление прогресса пользователя (H)
9) Аккаунт пользователя
	1) Создание аккаунта пользователя
	2) Авторизация пользователя в приложении
	
## Ограничения
1) Приложение управляется только с помощью мыши и клавиатуры
2) Пользовательский интерфейс на русском языке
3) Графическая часть приложения выполнена с помощью спрайтов
4) Серверная часть связывается с базой данных приложения
